Host {name}
    HostName {ip}
    User {user}
    Port {port}
    IdentitiesOnly yes
    IdentityFile ~/.ssh/{private_key}

Docker

docker pour windows : dockerToolBox

les containers utilisent les ressources des machines hôtes

chaque container a son propre :

  • Système de fichier
  • processus
  • appels mémoire
  • matériel
  • port reseau

les containers sont plus legers et flexibles que les vm

Docker engine est le programe qui permet la construction et l'execution des containers

Il utilise les namespace du noyau Linux ainsi que les groupes de contrôle

Commande pour lancer un container qui disparait quand celui-ci n'est plus utilisé

docker run --rm {image}

Pour trouver une image :

docker search {image}

Avantages de docker

  • séparation des préoccupation sysadmin / dev
  • Cycle de dev rapide
  • portabilité des app (d'un environnement à un autre)

Les images sont trouvables sur docker hub

liste des containers et leur état

docker info

pour accèder à un container en bash

docker run -ti {image}

#listing des container actif
docker ps -a 

#reveil du container
docker start {container_id}

#démarage du bash 
docker exec -ti {container_id} bash

#pour sortir
exit

#eteindre le container
docker stop {container_id}

pour lister les images presentes sur le pc

docker images

pour pull une image

docker pull {image}

docker run lance une nouvelle instance de l'image, si on veut juste relancer un container déjà initialisé, il faut faire docker start puis docker exec

Dockerfile

Il s'agit d'un fichier d'instruction d'installation d'un container qui doit respecter les bonnes pratiques

  • il doit etre le plus éphémère possible
  • n'installer que le necessaure
  • un container = un processus
  • limiter le nombre de couche
  • les arguments multi ligne doivent être trier par ordre alphabétique

FROM est la premiere instruction d'un dockerfile, toutes les images derivant d'une image originale

ARG {argument_name} permet de spécifié le nom d'un argument par exemple :

ARG PRJCT_NAME

RUN, CMD, ENTRYPOINT peuvent être définie selon une forme exec ou shell

RUN correspond à l'installation

RUN apt-get install ...

CMD spécifie la commande qui sera executée lors du démarrage de l'image

  • elle ne peut etre utilisée qu'une seule fois
  • si il y en a plusieurs, seul le dernier est utilisé
  • la commande et les paramètres peuvent etre surchargé par l'instruction

EXPOSE permet d'indiquer à docker que le container ecoute sur les ports indiqués

COPY copie des fichiers ou des dossiers locaux à l'interieur du container

il ne peut selectionner que des fichiers du dossier courant ou enfant donc pas de ../

pas possible d'utiliser des regex (*.php, ?, …)

le destinataire doit finir par un / si c'est un dossier ou plusieurs fichiers vont y être copiés
